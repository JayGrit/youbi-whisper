# ydbi-whisper 项目运行部署报告

## 1. 项目职责

`ydbi-whisper` 是 YouBi 流水线中的语音识别服务，对应阶段名为 `whisper`，数据库阶段表为 `whisper`。服务轮询 `status = 'ready'` 的 whisper 任务，读取 demucs 阶段产出的人声音频，使用 OpenAI Whisper 进行 ASR，并将识别文本与时间轴写入数据库。

在全链路阶段中的位置：

```text
downloader -> demucs -> whisper -> translator -> speaker -> combiner -> uploader
```

成功后会将下一阶段 `translator.status` 从 `pending` 推进到 `ready`。

## 2. 当前运行环境

- Python 版本：`>=3.12`
- 包管理：标准 Python packaging，项目定义在 `pyproject.toml`
- 启动入口：`ydbi-whisper = ydbi_whisper.main:main`
- 工作目录：默认 `/tmp/ydbi/whisper/<task_id>`，可通过 `YDBI_WHISPER_WORK_DIR` 覆盖
- 轮询间隔：`10` 秒
- Whisper 模型：`large-v3-turbo`
- 运行设备：`DEVICE = "auto"`，优先 `cuda`，其次 `mps`，最后 `cpu`
- 模型下载目录：`WHISPER_DOWNLOAD_ROOT = ""`，即使用 Whisper 默认缓存目录

服务需要能够访问：

- MySQL：当前配置指向远端 `youbi` 数据库
- MinIO：读取 demucs 上传的人声音频
- 本机或容器内可写的临时工作目录
- 可用的音频解码环境，通常需要 `ffmpeg`

- YouTube：`asr_language = "en"`
- Bilibili：`asr_language = "zh"`

### 运行身份

服务通过环境变量 `DEVICE` 标记当前运行机器或 operator：

- 未设置时默认为 `Macbook Air M4`

- `DEVICE` 会写入 `task` 和当前阶段表的 `operator` 字段

注意：这里的 `DEVICE` 环境变量用于数据库 operator；Whisper 推理设备由 `config.py` 中的 `DEVICE = "auto"` 常量控制。

## 4. 环境依赖

`pyproject.toml` 声明的 Python 依赖：

```text
minio>=7.2.0
mysql-connector-python>=9.0.0
openai-whisper
pydub
```

实际运行还需要：

- `torch`，由 `openai-whisper` 依赖或部署环境提供
- `ffmpeg`，供 Whisper 和 pydub 解码音频
- 足够的磁盘空间存放 Whisper 模型缓存
- GPU、Apple MPS 或 CPU 推理环境

MPS 运行时会禁用 word timestamps，以规避 MPS float64 DTW 问题；此时输出 segment 级时间轴仍保留，但单词级时间戳为空或不可用。

## 5. 数据流与产物

### 输入

服务从 `whisper` 查询 ready 任务，并合并 `video_info`。人声音频输入优先级：

1. 如果 demucs 阶段的 `operator` 与当前服务相同，并且 `audio_vocals_path` 存在，则使用本地文件
2. 否则使用 `audio_vocals_url` 从 MinIO 下载
3. 如果没有 URL 但本地路径存在，则使用本地路径
4. 如果都不可用，任务失败

下载目标默认：

```text
<work_dir>/<task_id>/media/audio_vocals.<suffix>
```

### 输出

识别结果不再写成本地 JSON 文件，而是写入数据库：

- `asr_segment`
- `whisper_word_timestamp`

`whisper` 成功后返回：

```text
asr_json_path = db://asr_segment/<task_id>
```

该字段会写入 `video_info.asr_json_path`。

## 6. 数据库表与状态推进

服务使用的核心表：

- `task`
- `whisper`
- `translator`
- `demucs`
- `video_info`
- `asr_segment`
- `whisper_word_timestamp`
- `service_heartbeat`

`asr_segment` 保存分段级 ASR 结果：

- `task_id`
- `item_index`
- `text`
- `start_time`
- `end_time`
- `speaker`

`whisper_word_timestamp` 保存词级时间戳：

- `task_id`
- `segment_index`
- `word_index`
- `text`
- `start_time`
- `end_time`

状态流转：

1. `whisper.status = ready`
2. 服务抢占任务，更新为 `running`
3. 成功后更新为 `success`
4. 推进 `translator.status = ready`
5. 失败后更新 `whisper.status = failed`，同时更新 `task.status = failed`

服务会自动创建或补充 ASR 表、`video_info` 表和 `operator` 字段。

## 7. 本地部署手册

### 1. 准备 Python 环境

```bash
cd /Users/hoshuuch/Money/YouBi/services/whisper
python3.12 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
python -m pip install -e .
```

### 2. 安装系统依赖

确保 `ffmpeg` 可用：

```bash
ffmpeg -version
```

macOS 可使用 Homebrew 安装：

```bash
brew install ffmpeg
```

### 3. 配置运行机器标识

```bash
export DEVICE="Macbook Air M4"
```

如需改变工作目录：

```bash
export YDBI_WHISPER_WORK_DIR="/tmp/ydbi/whisper"
```

### 4. 启动服务

```bash
ydbi-whisper
```

或：

```bash
python -m ydbi_whisper.main
```

首次启动并执行任务时，Whisper 会下载或加载 `large-v3-turbo` 模型，耗时取决于网络和缓存状态。

## 8. 运维检查

### 心跳检查

查看 `service_heartbeat` 中 `service_name = 'whisper'` 的对应设备列是否持续更新。

### 任务检查

关注：

- `whisper.status`
- `whisper.error_message`
- `task.current_stage`
- `video_info.audio_vocals_url`
- `video_info.asr_json_path`
- `asr_segment` 分段数量
- `whisper_word_timestamp` 词级时间戳数量

### ASR 结果检查

成功任务应满足：

```sql
SELECT COUNT(*) FROM asr_segment WHERE task_id = '<task_id>';
SELECT COUNT(*) FROM whisper_word_timestamp WHERE task_id = '<task_id>';
```

## 9. 常见问题

- 找不到人声音频：检查 `video_info.audio_vocals_path` 和 `audio_vocals_url`
- URL 来源不支持：目前只识别 YouTube 与 Bilibili
- 模型加载慢：确认模型缓存目录和磁盘空间
- MPS 下缺少词级时间戳：这是当前代码的兼容性处理
- ffmpeg 缺失：安装系统级 `ffmpeg`
- 跨机器执行找不到本地文件：确保 demucs 已上传 MinIO，或让同一 operator 机器处理

## 10. 当前风险与改进建议

- 数据库密码和 MinIO 密钥硬编码在源码中，建议改为环境变量或密钥管理
- 语言判断完全依赖 URL 来源，建议允许从任务配置覆盖
- `WHISPER_MODEL`、`WHISPER_DOWNLOAD_ROOT`、模型推理设备目前是源码常量，建议改为环境变量
- 当前没有 Dockerfile，跨机器部署需要额外保证 Python、torch、ffmpeg、模型缓存一致
